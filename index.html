    7. IP(Internet Protocol) represents the addressing scheme for devices to communicate over a network: every device has it's own individual one. In recent years, the number of computers, smartphones, etc. that are connected to the Internet has grown worringly fast, the problem of running out of IP addresses becoming a bigger one, due to the fact that IPv4 is using 32-bit addressing, therefore allowed for a (mere) total of 2^32 addresses (approx 4 billion). The answer of this problem was solved with a new version of IP: IPv6, which is using 128-bit addressing, resulting in a whopping 2^128 or 3.4*10^38 addresses, more than enough to satisfy our need for connectivity.
      Also, besides solving the address problem, IPv6supported other important changes meant to improve the IP protocol, such as:

      Discontinuing the Usage of NAT (Network Address Translation), therefore improving peer-to-peer performance.

      Improved Autoconfiguring ability, by offering simple, automatic configuration.

      Simplified Packet Header Structures, improving performance in routing by lowering the time required for header proccessing.

      Built-in Authentication and Privacy Support (IPSec): IPSec contains a set of cryptographic protocols used for ensuring secure data communication and key exchange.

      Easier Network Administration by making network renumbering able to execute automatically.

      Better multicast routing by offering a large address range, compared to IPv4.

      Superior mobility features offered by MIPv6

      Finally, by following the key design principles of its predecessor, compatibility between IPv6 and IPv4 is reliable: first of all, allowing nodes using both protocols to be used for sending a packet, and also this pro permits a smooth transition from the older version to the new one.
